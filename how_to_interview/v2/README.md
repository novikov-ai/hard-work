# Как выбесить интервьюера (2)

## Случай 1

На практике был случай, когда интервьюир пытался узнать как я буду траблшутить возникшую проблему с сервисом. При этом ответы, которые 
реально применяются в продакшне, не принимались, собеседник явно чего-то ждал. 

В тот момент я просто не догадался подумать в сторону того, чего хотел услышать собеседник, вместо дачи прямого ответа из практики.

## Случай 2

Однажды, отвечая на блиц-вопросы по Go (при этом отвечал неплохо), получил странный отказ после интервью. Интервьюир, как оказалось, ожидал ответы с определенным уклоном в домен, в котором они разрабатывали. 

На будущее: есть смысл в таких случаях прикладывать доп. умственные усилия, чтобы понять, что интервьюер хочет услышать по итогу и для чего он задает вопрос.
